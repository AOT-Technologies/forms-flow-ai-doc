RBAC -Role Based Access Control


formsflow introduces the ability for designers to set permissions for forms, catering to both designers and client users. This functionality provides fine-grained control over the visibility of forms based on user roles.

With this feature, designers using formsflow can define specific permissions with each form they create. They have the flexibility to determine which roles can view and interact with a particular form. By configuring these permissions, designers can ensure that only authorized individuals or user roles can access and work with the form.

Overall, this feature enhances the flexibility and control available to designers within formsflow, empowering them to manage permissions and determine who can interact with their forms. It ensures that the visibility of forms aligns with the assigned roles, creating a more secure and tailored environment for both designers and client users

  1. Log in as designer user and create a form then save it
  2. Attach the form with workflow
  3. select designer for the specific form.

    For designer permission

    • Case I : Accessible for all designers : default
    • Case II : Private(Only You) : If only the current designer need to access the form

      • Note: The designer of a form will always have access to the form, designed by him/her even though the option is Private(Only You).

    • Case III: Specific Designer Group : Give access to specific groups of designer

      Click on the Specific Designer Group option, and then click on Add button to display the list of groups available .In the same way, we can add specific client groups

      • Note: The designer of a form will always have access to the form, designed by him/her even though he/she is not a member of the specific group.

  4. select client for the specific form.

    For client permission

    • Case I : Accessible for all client : default
    • Case II : Specific Client Group : Give access to specific groups of clients

    5. NoteWhen choosing specific designer groups or client groups, the list contains all the groups available in Keycloak, including the designer groups, client groups, and reviewer groups. The designer has to choose the appropriate group required

  5. Publish and save the form for client users
